###### 为什么代理要用weak？
`weak`: 在`ARC`中，当带有`weak`修饰符的变量所引用的对象被销毁时，则将`nil`赋值给该变量。而代理的作用就是在当前对象中引用另一个对象，使用`weak`有两个好处：
* 当引用对象销毁时，`delegate`就会变成nil，这时候使用`delegate`去执行方法，不会`crash`，因为`Objective-C`允许像一个`nil`对象发送消息
* 当引用对象销毁时，delegate不会对引用对象持有强引用，即不会导致引用对象释放不了的情况，也可以避免循环引用。

##### 代理的delegate和dataSource有什么区别？
* delegate: 主要是存放当前类的对象所产生的行为
* dataSource: 主要是存放所有当前类所需资料相关的方法
以UITableView为例：
dataSource所包含的方法如下：

